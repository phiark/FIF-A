## 阶段 1：基线建立 (v0.0.0 ~ v1.0.0)

**时间跨度**：2024-11-09 ~ 2024-11-12

### 1.1 研究问题

1. 能否在 SST-2 噪声数据上建立可复现的基线？
2. FrictionLayer 的基本实现是否可行？
3. 能量信号是否与预测错误有关联？

### 1.2 技术方案

**核心创新**
- FrictionLayer：动态 μ、归一化拉普拉斯、η 衰减、1D 平滑
- 噪声条件化：训练时混合 clean+low+med/high 噪声
- 能量正则：`L = CE + λ·log(1+E)`

**实验配置**
- 模型：轻量 Transformer (4层, hidden=256, heads=4)
- 数据：GLUE SST-2 + SNLI
- 训练：5 epochs, batch=256, seed=42
- 正则：λ ∈ {0, 1e-4}

### 1.4 关键发现

#### ✅ 成功经验
1. **框架可行性确认**
   - FrictionLayer 基本实现成功，能量计算稳定
   - 统一 CLI 和数据管线建立
   - 结果可复现（固定 seed）

2. **能量信号存在**
   - 能量值确实随模型和噪声强度变化
   - Hybrid 模型产生明显的能量差异

#### ❌ 问题与挑战
1. **准确率下降**
   - Hybrid 在所有设置下均低于 Baseline
   - SNLI: -9.6 pt
   - SST-2 noisy: -1.5 ~ -4 pt

2. **能量相关性弱**
   - Pearson r ≈ 0 或弱负相关
   - 未实现"能量≈错误"的假设

3. **能量量级不一致**
   - 跨任务、跨噪声的能量刻度差异巨大
   - Hybrid 能量远高于 Baseline (10~100×)

### 1.5 结论与影响

**主要结论**
- ✅ 技术框架基本可行
- ❌ 准确率和能量相关性未达预期
- 📌 需要能量正则化和训练优化

**对后续版本的影响**
- v1.0.1：优化训练基础设施（DDP）
- v1.0.2：完善能量计算和 CLI
- v1.0.3：引入能量监控和正则

### 2.4 关键发现

#### ✅ 成功经验
1. **SST-2 性能改善**
   - Low噪声：Hybrid 超越 Baseline (+0.2~+2.6 pt)
   - Med/High噪声：Hybrid 保持竞争力
   - ECE 显著降低

2. **能量正则有效**
   - SST-2 中能量被压制到合理范围 (1~2)
   - 相关性出现正向迹象（虽然仍弱）

3. **性能优化成功**
   - kNN 向量化减少训练时间
   - Sortish batching 降低 padding 开销
   - Fixed-μ 提供速度-精度权衡选项

#### ❌ 持续问题
1. **SNLI 表现不佳**
   - Hybrid 持续低于 Baseline (-6~-8 pt)
   - 能量相关性仍然很弱 (r < 0.1)
   - 训练时间仍是 Baseline 的 2.9×

2. **能量量级不稳定**
   - SNLI Hybrid 能量暴走 (6.3k)
   - SST-2 Hybrid 能量被过度压缩 (<2)
   - Guard/watch 机制未有效触发

3. **相关性仍弱**
   - 所有设置下 Pearson r < 0.1
   - 能量无法有效预测错误

### 2.5 结论与影响

**主要结论**
- ✅ SST-2 任务初步成功
- ❌ SNLI 任务持续失败
- ⚠️ 能量正则效果有限

**诊断发现**（见 `docs/diagnostics/energy_confidence_gap.md`）
- 训练目标与评估指标错位（z-score vs raw）
- 能量定义未与分类难度耦合
- 尺度/长度主导，掩盖错误信号
- 正则权重易坍塌或形同虚设

**对后续版本的影响**
- v1.1.0：尝试 absolute 能量正则
- v1.2.0（规划中）：转向排名能量方案

### 3.4 关键发现

#### ❌ 失败分析
1. **跨任务行为不一致**
   - SST-2：能量被过度压缩 (2~10)，准确率下降
   - SNLI：能量完全失控 (4.6k)，准确率崩溃 (0.506)

2. **AUROC 提升微弱**
   - SST-2: +0.01~0.02
   - SNLI: +0.06
   - 但以准确率大幅下降为代价

3. **Absolute 正则失效**
   - 无法统一跨任务能量刻度
   - Guard 无法有效控制能量范围
   - λ=1e-4 在不同任务产生完全相反效果

4. **SNLI 基线异常**
   - Baseline 准确率仅 0.536（历史约 0.75+）
   - 怀疑数据加载/标签/配置问题

#### 📌 诊断结论
- ❌ Absolute 能量正则被判定为失败方案
- ⚠️ SNLI 基线需要修复
- 🔄 需要转向排序关系而非绝对刻度

### 3.5 决策与影响

**关键决策**
- 🔒 **冻结 v1.1.0**：不再调参或实验
- 🔄 **转向 v1.2.0**：采用 batch z-score + rank/margin 损失
- 🔧 **修复 SNLI**：优先排查基线问题

**对论文的影响**

## 阶段 4：排名能量方案 (v1.2.0 进行中)

**时间跨度**：2024-12-04 ~ 预计 2024-12-15
**状态**：🔄 **进行中**

### 4.1 研究问题

1. Batch z-score 能量 + rank/margin 损失能否改善相关性？
2. SNLI 基线能否恢复到正常水平 (>0.70)?
3. 排序关系优化是否优于绝对刻度优化？

### 4.2 技术方案

**核心创新**
- **能量归一化**：`E_norm = (E - μ_batch) / (σ_batch + ε)`
- **Rank 损失**：`L_energy = max(0, m + E_norm_correct - topk(E_norm_wrong))`
- **总损失**：`L = CE + λ · L_energy`
- **刻度对齐**：正则、评估、监控使用相同的归一化能量

**计划实验**
- Phase 1：SNLI 基线修复 (目标 acc > 0.70)
- Phase 2：干净数据集小网格 (λ ∈ {0, 0.1, 0.3}, margin=0.5)
- Phase 3：Noisy SST-2 扩展

### 5.6 关键经验总结

#### 成功经验 ✅
1. **框架设计**
   - 统一 CLI 和数据管线降低实验成本
   - 完整的结果产物便于追踪
   - 版本控制和文档同步保证可复现性

2. **性能优化**
   - 向量化批处理显著提升速度
   - 分阶段优化逐步解决瓶颈

3. **快速试错**
   - v1.1.0 失败后立即冻结，避免浪费资源
   - 失败案例提供宝贵洞察

#### 失败教训 ❌
1. **过早优化正则**
   - 应先确保基线稳定再引入复杂正则
   - SNLI 基线问题拖累整体进度

2. **忽视刻度一致性**
   - 训练、评估、监控使用不同能量定义
   - 导致优化方向与评估目标错位

3. **单一指标依赖**
   - 仅依赖 Pearson r 不足以判断成败
   - 应综合 AUROC/AURC/ECE 等多指标

#### 待验证假设 🔬
1. 排序关系 > 绝对刻度
2. Z-score 能量 > Raw 能量
3. Margin 损失 > 方差惩罚
4. 噪声条件化进入能量分支
